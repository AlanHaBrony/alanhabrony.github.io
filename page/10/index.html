Checking 404 within vs before the loop

2024-03-24 16:34
When I was modifying my script of scraping streetview, I found when the last iterate’ range become 4-figue numbers, the low speed dragged by the three for loops was magnified. The result was the iterate of requests became very slow at the speed of just 1 request per second.

This motivated me to improve the checking part of invalid urls. I examinated my code to find I put the if check inside the inner loop, for at first I just focused on the function of passing the invalid urls and then downloading the valid urls.

2024-03-24 16:48
This time I realized maybe the initialization of for loops itself also consumes time, so I put the if 404 check outside of the for loops by just use fixed parameter, for there are tiled images, one fragment’s existance can implicy all the fragments’.
And this time, the speed was truly improved by nearly ten times. Now it could post 10 requests per sencond. And the network measure doubled.
So I think this proves that the python’s mutiple for loops should contain as less as code as possible, for the 404 check doesn’t engage the iterate but it still slow down the overall speed.
